# EXECUTION_ROADMAP - quantum_x

## 阶段路线
| 阶段 | 目标 | 主要交付物 | 退出标准 |
|---|---|---|---|
| Phase 0 | 对齐与基线 | PRD/架构/体验地图/PDCA | 文档评审通过 |
| Phase 1 | 数据与研究底座 | 数据接入、特征、实验模板 | 基础策略可回测 |
| Phase 2 | 回测与风控 | 回测引擎、风控规则 | 回测与仿真一致 |
| Phase 3 | 实盘执行 | 订单链路、监控告警 | 小资金实盘稳定 |
| Phase 4 | 多策略与复制 | 策略编排、信号分发 | 跟单规则可审计 |

## 里程碑检查
- M1: 数据与研究环境稳定可用
- M2: 风控与执行链路打通
- M3: 多策略系统可运行
- M4: 跟单系统能力评审通过

## MVP 实施计划（基于假设）
### 临时假设
- MVP 仅做模拟盘（不接入实盘）
- 先聚焦单一资产域以快速验证（默认：虚拟币现货/永续）
- MVP 使用公开/免费数据源，付费数据后置
- 仅内部使用，不开放客户跟单

### 决策门禁
- 资产优先级（股票/期货/虚拟币）与目标合规地区
- 数据预算与延迟要求
- 交易通道与券商/交易所接入约束
- 风险限额与资金分配策略

### 工作流
1. 数据基线：接入契约、标准化、质量门禁
2. 特征仓库：在线/离线一致性与版本控制
3. 研究流水线：实验可复现与模型注册
4. 策略 MVP：基准策略 + 评审门禁
5. 执行一致性：统一订单契约的模拟盘
6. 风控与监控：事前/事中/事后风控 + 审计链路
7. MVP 评审：回测/模拟偏差与稳定性评估

### MVP 退出标准
- 策略通过泄漏/成本/压力测试门禁
- 回测与模拟偏差在阈值内
- 风控与审计链路端到端可用
